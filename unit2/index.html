Section 1
Data Cleaning
📖 What is Data Cleaning?
Data Cleaning (also called Data Wrangling or Data Scrubbing) is the process of fixing or removing incorrect, incomplete, duplicate, or corrupted data from a dataset before analysis.
🍽️ Analogy: Imagine you bought vegetables from a market. Before cooking, you wash them, remove rotten ones, and cut them properly. Data cleaning is exactly this — preparing your raw data before "cooking" (analysis)!

In real projects, data scientists spend 60–80% of their time just cleaning data. If your data is dirty, even the best model will give wrong answers. This is called: "Garbage In, Garbage Out!"

❓ 1.1 Handling Missing Values

Missing values appear as NULL, NaN, blank, or "—". They occur because of data entry errors, sensor failures, or unanswered survey questions.

🗑️
Method 1 — Delete the Row / Column
Remove rows with missing values. Simple but risky — you lose data. Only use if very few rows are missing (<5%).
✅ Use when: Very few missing values | ❌ Avoid when: Large portion of data is missing
📊
Method 2 — Fill with Mean
Replace missing value with the average of that column. Best for numerical data with no outliers.
📌 Example: Age missing → Fill with average age = (20+21+19)/3 = 20
📍
Method 3 — Fill with Median
Replace with the middle value when sorted. Best for data with outliers (outliers don't affect median).
📌 Example: Salary data has outliers → Use median salary instead of mean
🔢
Method 4 — Fill with Mode
Replace with the most frequent value. Best for categorical (text) data.
📌 Example: Grade column missing → Fill with most common grade "B"
🔮
Method 5 — Forward/Backward Fill
Use the previous or next value to fill the gap. Used in time series data (e.g., daily temperatures).
📌 Example: Monday temp=30°, Tuesday temp=?, Wednesday temp=32° → Fill Tuesday with 30° or 32°

📊 1.2 Handling Outliers

📖 What is an Outlier?
An outlier is a data point that is very different from the rest — either much too high or much too low. It can distort your analysis badly.
📌 Example
Marks of 5 students: 55, 60, 58, 62, 998
Mean with outlier = (55+60+58+62+998)/5 = 246.6 ← WRONG picture!
Mean without outlier = (55+60+58+62)/4 = 58.75 ← Accurate!

🔍 IQR Method — Most Common Outlier Detection

IQR = Q3 − Q1
Lower Bound = Q1 − 1.5 × IQR
Upper Bound = Q3 + 1.5 × IQR
Any value outside these bounds = Outlier
📌 Worked Example
Data: 10, 12, 14, 15, 16, 17, 18, 90
Q1 = 13, Q3 = 17.5, IQR = 4.5
Lower = 13 − (1.5×4.5) = 13 − 6.75 = 6.25
Upper = 17.5 + 6.75 = 24.25
→ 90 is outside [6.25, 24.25] → 90 is an OUTLIER! ✓

🛠️ How to Handle Outliers?

Remove them
Delete the outlier row if it's clearly a data entry error (e.g., marks = 998).
Cap / Clip them
Replace with the upper/lower bound value. E.g., cap all values above 24.25 to 24.25.
Keep them!
If the outlier is real (e.g., Sachin's 100 centuries), keep it — it's meaningful data!
Transform the data
Apply log transformation to reduce the effect of extreme values on the analysis.

Section 2
Data Transformation
📖 What is Data Transformation?
Data Transformation converts data from one format or scale into another, making it suitable for machine learning models and statistical analysis.
🤔 Why transform? ML algorithms are sensitive to scale. If Age (20–25) and Salary (20,000–80,000) are in the same model without transformation, Salary will dominate just because its numbers are bigger — not because it's more important!

📏 2.1 Normalization (Min-Max Scaling)

Normalization scales all values to a range of 0 to 1. The smallest value becomes 0, the largest becomes 1, and all others fit in between.

X_norm = (X − X_min) / (X_max − X_min) Result is always between 0 and 1
📌 Worked Example — Student Marks
Marks: 40, 60, 80, 100
X_min = 40, X_max = 100

Normalise 40: (40−40)/(100−40) = 0/60 = 0.00
Normalise 60: (60−40)/(100−40) = 20/60 = 0.33
Normalise 80: (80−40)/60 = 40/60 = 0.67
Normalise 100: (100−40)/60 = 60/60 = 1.00

📐 2.2 Standardization (Z-Score)

Standardization converts values so that the data has mean = 0 and standard deviation = 1. Unlike normalization, the result is NOT bounded to 0–1. Values can be negative.

Z = (X − μ) / σ μ = Mean of data  |  σ = Standard Deviation  |  Z = Z-score
🆚 Normalization vs Standardization — When to use which?
Use Normalization when you know data has a bounded range and no extreme outliers.
Use Standardization when data has outliers or you don't know the range. It's more robust.
📌 Worked Example
Data: 2, 4, 6, 8, 10  |  Mean (μ) = 6  |  Std Dev (σ) = 2.83

Z-score of 2: (2−6)/2.83 = −1.41 (below average)
Z-score of 6: (6−6)/2.83 = 0.00 (exactly average)
Z-score of 10: (10−6)/2.83 = +1.41 (above average)

🔤 2.3 Encoding Categorical Data

ML models work with numbers only. Text categories like "Male/Female" or "Good/Bad" must be converted to numbers. This is called Encoding.

How it works: Each unique category gets a number: 0, 1, 2, 3...

⚠️ Problem: The model may think Grade "F" (3) is greater than Grade "A" (0), which is wrong!
✅ Use when: Categories have a natural order (e.g., Low=0, Medium=1, High=2)
How it works: Create one binary column (0 or 1) for each category value.

Section 3
Data Integration
📖 What is Data Integration?
Data Integration is the process of combining data from multiple, different sources into a single, unified view for analysis.
🏥 Real Example: A hospital has patient data in 3 different systems: Registration system (name, age), Lab system (test results), Pharmacy system (medicines given). Data Integration combines all 3 into one complete patient record!

🔧 Methods of Data Integration

📋
Manual Integration
Copy-paste data from different files into one Excel sheet. Slow and error-prone but simple.
⚙️
ETL (Extract, Transform, Load)
Automated pipeline: Extract from sources → Transform/clean → Load into target database. Used in industry.
🗄️
Data Warehouse
A central database that stores integrated data from multiple sources. Used for business reporting.
🌐
API Integration
Automatically fetch and combine data from online sources using APIs (e.g., combining weather + sales data).

🧩 Database Joins — The Core of Integration

When combining two tables, we use Joins — a concept from SQL databases. Think of it like connecting two spreadsheets using a common column (like Student ID).

Join TypeWhat it doesAnalogy
INNER JOINOnly rows that match in BOTH tablesOnly mutual friends
LEFT JOINAll rows from left table + matching from rightAll of yours + common
RIGHT JOINAll rows from right table + matching from leftAll of theirs + common
FULL JOINAll rows from both tablesEveryone, from both sides
📌 Visual Example — INNER JOIN
Table A — Students:
101-Rahul | 102-Priya | 103-Arjun

Table B — Marks:
101-85 | 102-72 | 104-90

INNER JOIN result (only matching IDs 101, 102):
101-Rahul-85 | 102-Priya-72
→ ID 103 (no marks) and ID 104 (no name) are excluded.

⚠️ Challenges in Data Integration

🔴 Schema Conflicts
Same data stored differently: One table says "DOB", another says "Date_of_Birth". Need to standardize column names.
🟡 Duplicate Records
Same person appears in two systems with slightly different names. "Rahul Sharma" vs "R. Sharma".
🟣 Data Conflicts
Two sources have contradictory values for the same entity. Which source is correct?
🔵 Different Formats
Dates in different formats, currencies in different units. Needs transformation before merging.

Excel for Data Handling

Microsoft Excel is the most widely used data tool in the world — especially in Indian businesses. Before you learn Python or SQL, mastering Excel gives you a huge practical advantage!

💼 Industry Fact: More than 750 million people use Excel worldwide. In India, almost every office job requires Excel skills. Data Analysts at banks, hospitals, and retail companies use Excel every single day!

🔧 Essential Excel Functions

📐 Basic Statistical Functions
fx=SUM(B2:B10)
→ Adds all values in cells B2 to B10. Example: Total marks of 9 students = 630
fx=AVERAGE(B2:B10)
→ Calculates mean. Example: Average marks = 70
fx=COUNT(B2:B10)
→ Counts cells with numbers. Example: 9 students have marks entered = 9
fx=MAX(B2:B10)  |  =MIN(B2:B10)
→ Highest mark = 95, Lowest mark = 42
fx=MEDIAN(B2:B10)
→ Middle value when sorted. Useful when data has outliers. Example: 72
🔀 IF Function — Conditional Logic
Used to make decisions: "IF something is true, show this, ELSE show that."
fx=IF(B2>=40, "Pass", "Fail")
→ If marks ≥ 40, show "Pass", otherwise show "Fail"
fx=IF(B2>=75,"A", IF(B2>=60,"B", "C"))
→ Nested IF: Grade A if ≥75, Grade B if ≥60, Grade C otherwise

🔢 COUNTIF & SUMIF — Conditional Count/Sum
Count or sum cells that meet a specific condition.
fx=COUNTIF(C2:C10, "Pass")
→ Count how many cells say "Pass". Example: 7 students passed
fx=SUMIF(C2:C10, "Pass", B2:B10)
→ Sum marks of only students who "Pass". Example: Total passing marks = 510
fx=COUNTIF(B2:B10, ">=75")
→ Count students with marks ≥ 75. Example: 3 students got A grade
🔍 VLOOKUP — Searching in Tables
Look up a value in the first column of a table and return a value from the same row. Like a search engine for your spreadsheet!
fx=VLOOKUP(lookup_value, table_range, col_num, 0)
→ Parameters: What to search | Where to search | Which column to return | 0 = exact match
fx=VLOOKUP(105, A2:C10, 2, 0)
→ Find Student ID 105 in column A, and return the value from column 2 (Name) = "Anjali"
🎯 Real Use Case
You have a product ID in one sheet and prices in another. VLOOKUP finds the ID in the price list and returns the correct price automatically — no manual searching!
✨ More Useful Functions for Data Cleaning in Excel
fx=TRIM(A2)
→ Removes extra spaces. Fixes " Rahul " → "Rahul"
fx=UPPER(A2)  |  =LOWER(A2)  |  =PROPER(A2)
→ "rahul" → RAHUL | rahul | Rahul — Fixes case inconsistencies!
fx=ISNUMBER(A2)  |  =ISBLANK(A2)
→ Check if a cell has a number or is empty. Returns TRUE or FALSE.
fx=CONCATENATE(A2, " ", B2)
→ Join cells: "Rahul" + " " + "Sharma" = "Rahul Sharma"
fx=SUBSTITUTE(A2, "male", "Male")
→ Replace text inside a cell. Fixes encoding inconsistencies automatically.
